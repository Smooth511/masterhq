## Finding 1 — mintsysadm.py is a grub write-back trap

**File:** `/usr/lib/python3/dist-packages/mintsysadm/mintsysadm.py`
(or equivalent path on installed system)

The legitimate mintsysadm script has been injected with a **duplicate save function block**
that fires after the real one. Identified by:
- Chinese character line `什 吧二0：` mid-code (injected marker)
- Duplicate `GRUB_TIMEOUT_STYLE`, `GRUB_TIMEOUT`, `GRUB_CMDLINE_LINUX_DEFAULT` definitions
- The save function expands `$GRUB_CMDLINE_LINUX_DEFAULT` — rootkit's cage flags already
  live in that variable, so every time user opens System Admin and saves, they write
  rootkit's flags back into grub themselves

**Effect:** Any attempt to strip cage flags via the GUI tool silently restores them.

**Headshot:** Replace with clean version from `github.com/linuxmint/mintsysadm` before
touching grub via GUI. The injected second block is the actual target.
