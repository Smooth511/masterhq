Wipe the partition, then you're reflashing:

bash
# First unmount it
sudo umount /mount/2/3/4/5/6/7/8

# Wipe partition 7 completely (zeros)
sudo dd if=/dev/zero of=/dev/sda7 bs=4M status=progress

# Or if it's nvme:
sudo dd if=/dev/zero of=/dev/nvme0n1p7 bs=4M status=progress
Check which device it actually is first:

bash
lsblk
That'll show you the exact device name for partition 7. Replace /dev/sda7 or /dev/nvme0n1p7 with whatever lsblk shows.

Once wiped, flash the fresh ISO to it:

bash
sudo dd if=/path/to/fresh.iso of=/dev/sda7 bs=4M status=progress conv=fsync
status=progress so you can see it working. conv=fsync to make sure it's fully written before dd exits.

If you're wiping the WHOLE DISK not just partition 7 — use the disk device without the partition number (/dev/sda not /dev/sda7, or /dev/nvme0n1 not /dev/nvme0n1p7).

Depends what you want:

Just wipe it (no file needed):

bash
sudo dd if=/dev/zero of=/dev/sdd bs=4M status=progress conv=fsync
/dev/zero = infinite zeros. Writes straight to disk. No file involved.

Flash an ISO onto it (need the .iso file):

bash
sudo dd if=/path/to/whatever.iso of=/dev/sdd bs=4M status=progress conv=fsync
You need the file because that's the data being written. dd copies bytes from if= to of=. No file = no data to copy.

What you were doing wrong: You mounted sdd1 to a directory, then tried to dd the directory back onto sdd1. dd doesn't work on directories — it reads raw bytes from files or devices. The mount point is just where the filesystem appears, it's not the raw data.

If you did it differently before — you probably used if=/dev/sdc or if=/home/whoever/ubuntu.iso or similar. That's the correct approach. What changed today is you tried to use the mounted directory path instead.

Quick version — just nuke it:

bash
sudo umount /mount/2/3/4/5/6/7 && sudo dd if=/dev/zero of=/dev/sdd bs=4M status=progress conv=fsync
