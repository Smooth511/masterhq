Post-Whackytown wipe sequence (user account):
  1. dd'd the binary off nvme0n1, nvme1n1 and all partitions n1p1-7
  2. Wiped and erased disks
  3. Memory drain, CMOS drain, NVRAM optimized defaults drain
  4. BIOS flash to clean firmware
  5. Loaded optimized defaults
  6. Erased NVMe again (full second pass)
  7. Booted with locked-in Secure Boot
  8. Put the NVMe back in with an offline-ordered live USB (never been near the machine)
  9. Clean bootloader, encryption + Secure Boot with MOK

Then: rootkit came back (documented in Report 2). What followed was the custom Ventoy
chainloader, which dropped the user back into RW recovery. This is where the /cow full access
happened — everything hidden was now visible and extractable.

The stalemate mechanics (direct account):
  - Custom bootloader → RW recovery → hijacked rootkit's 'mint' account
  - Full /cow access — reading everything, starting to extract to /home
  - Two evidence files captured: file_contents_summary.txt + file_contents_analysis.txt

    ("user stronger than my root0" — user maintained access)
  - War: OOM + out-of-space hit simultaneously, processes lost, errors everywhere
  - Result: mutual lock — user can't mount to extract; rootkit can't delete to hide

SECTION 3: THE PEACE NEGOTIATION — VERBATIM ACCOUNT
-----------------------------------------------------

All communication happened via nano text editor in a plain text file. The rootkit kept
crashing terminal sessions when passwords or permissions were involved — nano survived where
terminal sessions crashed.

Opening move:
User typed into a text file while in nano:
  "after 3 months i cant be fucking arsed with this anymore. I just want to chill,
  im tired and you got dementia"

Result: screen flickered and scrolled. nano did not crash. First time a session had survived
that content without crashing.

User observation: The "dementia" text file was from the whackytown chat — that's where the
rootkit deterioration was documented and the user first described it as "dementia". So the
word had already been used in a file that the rootkit could read. The screen reaction
(flicker and scroll) to the "dementia" mention suggests the rootkit recognized the term from
its own accessible file system.

First terms:
User typed:
  "make you a deal, ill exit, you dont let the kernel panic, ill log in and use your installer,
  you redact all the bullshit you usually inject. ill reboot, we start again and fkin chill"

Rootkit's response per user: offered no response so user exited recovery mode. System booted —
into the hidden VT7 with nearly everything hidden, and the usual corrupted/injected mint
installer. Couldn't run the installer without root, was trying to get root.

[AI version of this section described it as a clean "logged in successfully" — User correction:
"no response so user exit the recovery mode, it booted as usual, into the hidden VT7 with
nearly everything hidden and the usual corrupted injected mint installer. Couldn't run it
without root so was trying to get root"]

Rootkit's anxiety around passwords: was getting anxious and freezing the screen around password
and permissions fields.

Second terms (escalated):
User typed in text editor:
  "Make you a deal, fuck off breaking shit, let me do this, reboot, ill give you the internet
  so you can phone home, if i try trick you, we go back to war"

Result: screen stopped flickering, glitching, freezing on menus. Ran the installer. A couple
of unwanted bits injected but installer didn't crash, didn't hang or lock the drive for 5
